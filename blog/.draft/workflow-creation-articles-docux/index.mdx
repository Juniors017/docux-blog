# Workflow de Création d'Articles : 3 Méthodes Ultra-Efficaces

:::tip 🎯 Objectif du Guide
À la fin de ce tutorial, vous maîtriserez **3 workflows** complémentaires pour créer des articles optimisés SEO en **30 secondes** au lieu de 30 minutes !
:::

## 🛠️ Matériel Nécessaire

- ✅ **Node.js 16+** installé sur votre système
- ✅ **VS Code** avec support MDX
- ✅ **Workspace Docux Blog** configuré
- ✅ **Terminal** (PowerShell, Bash, etc.)

## 📋 Méthode 1 : Scripts CLI Automatisés

### Étape 1.1 : Script Interactif Complet

**Utilisation** : Mode guidé avec questions détaillées

```bash
node scripts/create-article.js
```

**Processus** :
1. Sélection du type de schéma dans une liste
2. Configuration automatique du frontmatter
3. Génération du contenu de base
4. Validation et création du fichier

**Temps** : 2 minutes | **Niveau** : Débutant

### Étape 1.2 : Script Rapide Express

**Utilisation** : Génération instantanée en ligne de commande

```bash
# Syntaxe
node scripts/quick-article.js [type] [titre]

# Exemples pratiques
node scripts/quick-article.js blog "Mon Nouvel Article"
node scripts/quick-article.js tech "Guide React Avancé"
node scripts/quick-article.js howto "Comment Installer Docker"
```

**Temps** : 30 secondes | **Niveau** : Tous niveaux

### Types Supportés

| Type | Description | Exemple d'Usage |
|------|-------------|-----------------|
| `blog` | Article standard | News, opinions |
| `tech` | Guide technique | Tutorials, docs |
| `howto` | Étapes détaillées | Instructions |
| `faq` | Questions/réponses | Support |
| `collection` | Pages de collection | Listes |
| `software` | Documentation logiciel | Apps |
| `course` | Formation | Cours |

## 📋 Méthode 2 : Snippets VS Code Intelligents

### Étape 2.1 : Activation des Snippets

Les snippets sont **pré-configurés** dans `.vscode/docusaurus-snippets.json`

### Étape 2.2 : Utilisation des Templates

1. **Créer** un nouveau fichier `.mdx`
2. **Taper** le préfixe + `Tab` :
   - `doc-blog` → Article de blog standard
   - `doc-tech` → Guide technique complet
   - `doc-howto` → Tutorial étape par étape

3. **Naviguer** avec `Tab` entre les champs
4. **Compléter** les informations personnalisées

### Étape 2.3 : Variables Automatiques

- **Date courante** : `${CURRENT_YEAR}-${CURRENT_MONTH}-${CURRENT_DATE}`
- **Navigation intuitive** : Passage automatique au champ suivant
- **Suggestions intelligentes** : Auto-complétion activée

**Temps** : 1 minute | **Niveau** : Intermédiaire

## 📋 Méthode 3 : Workflow Hybride Optimal

### Étape 3.1 : Stratégie par Contexte

**Création Ultra-Rapide** (actualités, idées spontanées) :
```bash
node scripts/quick-article.js blog "Breaking News React 19"
```

**Création Détaillée** (guides, tutorials) :
```bash
node scripts/create-article.js
# → Configuration interactive personnalisée
```

**Création Personnalisée** (contenu unique) :
1. Nouveau fichier `.mdx`
2. `doc-tech` + `Tab` 
3. Customisation manuelle avancée

### Étape 3.2 : Workflow Recommandé par Type

#### Articles de Blog Quotidiens
```bash
# Batch creation pour une semaine
node scripts/quick-article.js blog "Lundi: Actualités JS"
node scripts/quick-article.js blog "Mercredi: Tips React"
node scripts/quick-article.js blog "Vendredi: Outils du Dev"
```

#### Série de Guides Techniques
```bash
# Mode interactif pour personnalisation
node scripts/create-article.js
# → Sélectionner "TechArticle"
# → Configurer série et niveau
```

#### Documentation Complète
```bash
# Combinaison scripts + snippets
node scripts/quick-article.js faq "FAQ API"
node scripts/quick-article.js tech "Guide API Endpoints"  
# + Personnalisation manuelle avec snippets
```

## 🔧 Configuration et Personnalisation

### Personnaliser les Templates Scripts

**Fichier** : `scripts/create-article.js`

```javascript
// Ligne 15 - Configuration par défaut
const DEFAULT_CONFIG = {
  author: "votre-nom",        // 🔧 Changez ici
  baseImagePath: "/img/",     // 🔧 Votre chemin
  defaultTags: ["votre-tag"], // 🔧 Tags par défaut
  language: "fr-FR",          // 🔧 Langue
  copyright: "Votre Nom"      // 🔧 Copyright
};
```

### Personnaliser les Snippets VS Code

**Fichier** : `.vscode/docusaurus-snippets.json`

```json
{
  "Mon Template Custom": {
    "scope": "mdx",
    "prefix": "doc-custom",
    "description": "Mon template personnalisé",
    "body": [
      "---",
      "title: \"${1:Titre}\"",
      "// ... votre configuration"
    ]
  }
}
```

### Optimiser VS Code pour MDX

**Fichier** : `.vscode/settings.json`

```json
{
  "editor.snippetSuggestions": "top",
  "[mdx]": {
    "editor.quickSuggestions": true,
    "editor.formatOnSave": true
  }
}
```

## 🐛 Dépannage Rapide

### Erreurs Communes

**"Directory already exists"**
```bash
# Solution : Supprimer le dossier existant
Remove-Item "blog/2025/08/29/article-existant/" -Recurse -Force
```

**"Invalid schema type"**
```bash
# Types valides seulement
node scripts/quick-article.js tech "Mon Guide"  # ✅
node scripts/quick-article.js invalid "Test"    # ❌
```

**Snippets non disponibles dans VS Code**
```bash
# Vérifier l'extension MDX
code --install-extension unifiedjs.vscode-mdx
```
