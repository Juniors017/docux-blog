# FAQ SEO Docusaurus - Questions Fréquentes

:::info À propos de cette FAQ
Cette page répond aux **questions les plus courantes** sur l'implémentation SEO avec Docusaurus. Elle utilise le **schema FAQPage** pour optimiser l'affichage dans les résultats Google.
:::

## 🚀 Installation et Configuration

### Comment installer le composant SEO sur Docusaurus ?

**Réponse :** L'installation se fait en 3 étapes simples :

1. **Créez le composant** `src/components/Seo/index.jsx`
2. **Intégrez globalement** dans `src/theme/Layout/index.js`
3. **Configurez les métadonnées** via frontmatter

```jsx title="src/theme/Layout/index.js"
import React from "react";
import Layout from "@theme-original/Layout";
import Seo from "@site/src/components/Seo";

export default function LayoutWrapper(props) {
  return (
    <>
      <Seo />
      <Layout {...props} />
    </>
  );
}
```

### Quels sont les prérequis système ?

**Réponse :**

- **Docusaurus 3.8+** (versions antérieures non testées)
- **React 18+** pour les hooks modernes
- **Node.js 18+** pour le build
- **Navigateur moderne** pour le développement

## 📊 Schémas JSON-LD

### Quels schémas JSON-LD sont supportés ?

**Réponse :** La version 2.1.4 supporte :

| Schéma             | Usage              | Configuration                     |
| ------------------ | ------------------ | --------------------------------- |
| **BlogPosting**    | Articles de blog   | `schemaTypes: ["BlogPosting"]`    |
| **TechArticle**    | Contenu technique  | `schemaTypes: ["TechArticle"]`    |
| **HowTo**          | Guides pratiques   | `schemaTypes: ["HowTo"]`          |
| **CollectionPage** | Listes/Collections | `schemaTypes: ["CollectionPage"]` |
| **FAQPage**        | Pages FAQ          | `schemaTypes: ["FAQPage"]`        |
| **BreadcrumbList** | Navigation         | Automatique                       |

### Comment configurer des schémas multiples ?

**Réponse :** Utilisez la **configuration explicite v2.1.4** :

```yaml
---
title: "Mon Article"
schemaTypes: ["TechArticle", "BlogPosting"] # Double schéma
proficiencyLevel: "Advanced"
programmingLanguage: "JavaScript"
---
```

**Avantages :**

- ✅ **Contrôle total** : Vous décidez exactement
- ✅ **Prédictibilité** : Résultats garantis
- ✅ **Performance** : Pas de détection automatique
- ✅ **Rich Results** : Double exposition Google

### Le SEO fonctionne-t-il avec le build statique ?

**Réponse :** **Oui, parfaitement !** Le composant est optimisé pour la génération statique :

- ✅ **SSG-friendly** : Métadonnées générées au build
- ✅ **Pas de JavaScript client** requis pour le SEO
- ✅ **Performance maximale** : HTML statique avec métadonnées
- ✅ **Indexation rapide** : Google accède directement aux données

## 🧪 Test et Validation

### Comment tester les Rich Results Google ?

**Réponse :** Utilisez les outils officiels Google :

1. **[Rich Results Test](https://search.google.com/test/rich-results)** : Test immédiat
2. **[Google Search Console](https://search.google.com/search-console)** : Suivi en production
3. **[Schema.org Validator](https://validator.schema.org/)** : Validation technique

**Workflow recommandé :**

```bash
# 1. Test local
npm run start

# 2. Build et déploiement
npm run build
npm run serve

# 3. Test Google après déploiement
```

### Que faire si les schémas ne sont pas détectés ?

**Réponse :** Vérifications à effectuer :

1. **Frontmatter correct** : `schemaTypes: ["BlogPosting"]`
2. **Intégration Layout** : Composant SEO dans Layout global
3. **Build complet** : `npm run build` puis test
4. **URLs publiques** : Google ne teste que les URLs accessibles
5. **Patience** : 24-48h pour l'indexation Google

## 🆕 Nouveautés v2.1.4

### Quelles sont les nouveautés de la version 2.1.4 ?

**Réponse :** **Révolution des schémas explicites !**

- 🎯 **Configuration explicite** : `schemaTypes` dans frontmatter
- 🏆 **Hiérarchie priorités** : Explicite > Auto-détection
- 🔗 **Fragments URLs** : #blogposting, #techarticle pour éviter doublons
- ⚡ **Performance** : Bypass détection automatique
- 🛡️ **Prédictibilité** : Résultats 100% contrôlés

### Comment migrer vers la v2.1.4 ?

**Réponse :** Migration simple en 3 étapes :

```yaml
# AVANT (v2.1.3)
---
title: "Mon Article"
# Détection automatique uniquement
---
# MAINTENANT (v2.1.4)
---
title: "Mon Article"
schemaTypes: ["TechArticle", "BlogPosting"] # 🎯 Explicite

---
```
